🔬 추천 연구주제

1. "Pre-VLA + CrossVLA + EvoScene/SOMA" 를 backbone으로 한 VLA Runtime-Verifiable Architecture Stack

인사이트 #1의 7편을 한 framework로 묶는 라인이에요. Pre-VLA의 preemptive runtime verification을 backbone으로, CrossVLA의 DPO surrogate(discrete-token AR + continuous-action flow matching 양쪽)와 EvoScene-VLA+SOMA의 persistent action-updated scene state를 한 stack에 stacking하면 "VLA의 deployment-time architecture가 한 결정으로 결정되는 통합 stack"이 만들어집니다. 구체적 단계: (1) 우리 lab manipulation pipeline VLA에 Pre-VLA의 action validity assessment + world model rollout filtering을 plug-in해 'physical execution 전 verification gate' 구축, (2) DPO post-training은 CrossVLA의 surrogate log-prob estimator로 우리 flow matching policy(예: π-0.5 계열)에 직접 적용, (3) chunked control 사이의 scene belief는 EvoScene-VLA의 action-updated state + SOMA의 movable head camera spatial memory를 결합해 'visual frustum 너머 reasoning'까지 동시 처리, (4) 어제 manipulation diagnostics(DEFLECT · RoVLA · PAPO) 측 latency 측정과 합쳐 runtime overhead < 50ms 보장. ICRA·CoRL·CVPR 한 사이클의 주력 후보고, 'VLA runtime-verifiable architecture'라는 framing은 아직 paper-level 정공으로 본 곳이 없어 빈자리도 명확합니다.

2. "SegCompass + CEDAR + Ablate-to-Validate"를 backbone으로 한 VLM Mechanism-Aware Perception Stack

인사이트 #2의 SAE 기반 mechanistic 답 6편을 우리 lab의 VLM 기반 perception stack에 직접 활용하는 라인. 구체적 단계: (1) SegCompass의 SAE-based interpretable alignment를 우리 stack의 reasoning segmentation backbone(예: Qwen3-VL · LLaVA-family)에 적용해 'latent query alignment의 black box를 SAE로 deconvolve'한 baseline 구축, (2) CEDAR의 dimensionality-preserving disentanglement로 기존 representation의 geometry 손상 없이 sparse decomposition, (3) Ablate-to-Validate의 mechanistic 진단 원칙을 우리 모델의 'visual thinking token이 실제로 reasoning에 쓰이는가' 점검 protocol로 채택, (4) Visual-Advantage On-Policy Distillation으로 우리 VLM의 'visual input reliance'를 deployment-time에 직접 강화, (5) GeoWeaver의 token별 geometric evidence weighting을 spatial reasoning task에 추가. 이 stack은 단순 평가 framework이 아니라 우리 lab의 VLM-based perception layer 자체의 mechanism-aware monitoring/training 인프라가 됩니다. 🌟 오늘의 must-read

🌟 Pre-VLA: Preemptive Runtime Verification for Reliable Vision-Language-Action and World-Model Rollouts

핵심 주장: 큰 VLA + 생성형 world model 두 stack 모두 'low-quality action이 physical failure 또는 misleading WM rollout'을 일으키는 결정적 약점이 있다는 걸 정면으로 짚고, action validity assessment를 physical execution / WM rollout 직전에 preemptive하게 수행하는 'unified runtime verification architecture' 첫 시도예요. 어제 driving VLA 4편의 'visual evidence usage 진단' 라인이 한 일 만에 정확한 architecture 측 정공 답으로 등장한 결이라, 다음 분기 VLA architecture의 표준 컴포넌트가 될 가능성이 큽니다.

방법의 핵심: VLA action a_t와 world model rollout candidate ŝ_{t+1:t+H}에 대해 'physical feasibility + WM consistency'의 dual validity score V(a_t, ŝ)를 학습된 verifier로 계산. V < τ면 action을 reject하고 fallback policy 또는 alternative sample로 우회 — runtime overhead는 single forward pass 수준. Action 측은 reachability/collision 측 implicit constraint, WM 측은 'rollout이 consistent하면서 action이 realize 가능한가'의 joint check.

핵심 결과: 정량 수치는 abstract에 명시 없으나 'physical execution failure 감소 + WM rollout의 redundant rendering cost 감소' 두 축에서 동시 개선을 주장. 어제 manipulation VLA 5축(DEFLECT · RoVLA · PAPO · ZPRL · SafeAlign)의 'closed-loop deployment brittleness'에 대한 정확한 layer 측 답 — 'training 측 robustness' 너머 'inference-time gate'로 problem layer가 옮겨간 첫 unified stack입니다.

약점·한계: (1) verifier 자체의 training data가 필요한데, 'low-quality action'을 정의하는 ground truth가 domain-specific해서 generalization에 의문이 남아요. (2) WM rollout consistency check가 verifier의 false negative로 valid action을 reject할 risk(over-conservative). (3) Runtime overhead가 abstract에 정량 명시 없는데, 50ms control loop 안에 들어가야 실용성이 보장됩니다. (4) Manipulation/driving 각 도메인에 verifier가 별도 학습돼야 하는가, 통합 가능한가의 cross-domain transfer가 미진단.

우리 랩 영향: 추천 연구주제 #1의 'VLA Runtime-Verifiable Architecture Stack'의 backbone 컴포넌트로 직접 채택 가능. 우리 manipulation pipeline의 VLA 측에 Pre-VLA verifier를 plug-in하면 어제 manipulation 5축의 'deployment brittleness' 정공 답이 stack 측에서 한 layer 위에 얹힙니다. 한 분기 내 ICRA·CoRL paper로 응결될 timing.
