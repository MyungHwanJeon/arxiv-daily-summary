E2E driving이 본격적으로 VLA paradigm으로 흡수되는 변곡점입니다.

한편 부상 중인 미니토픽 두 가지. 첫째, VLA "inference engine" 라인이 오늘은 paradigm-축으로 한 단계 더 들어가요 — 어제까지는 "thin-layer intervention 6종(A3, GridS, Premover 등)"이라는 architectural sub-line이었다면, 오늘은 AR-VLA vs Diffusion-VLA 두 paradigm을 동시에 가속하는 라인이 등장합니다. BlockVLA는 autoregressive VLA를 block-diffusion finetuning으로 가속, Realtime-VLA FLASH는 diffusion VLA를 speculative inference로 가속해요. 거기에 AttenA+가 action inequality(모든 action을 동일 중요도로 다루는 가정)를 rectify하고, FrameSkip이 dense demonstration의 informative frame만 골라 학습하고, D-VLA가 high-concurrency distributed async RL을 VLA에 가져옵니다. 둘째, 3DGS의 응축 axis 전환입니다 — pretty rendering 단계가 끝나고 "in-the-wild deployment kit" 시대로 옮겨가는 결이 명확해졌어요. HarmoGS(in-the-wild conflict-aware gradient harmonization), RoSplat(robust generalizable feed-forward), BlitzGS(city-scale distributed), Z-Order Transformer(latency-friendly feed-forward), GuardMarkGS(ownership tracing + edit deterrence), Real2Sim(physics-driven editable GS for AD) — 6편이 같은 결로 모이는데, 평가축이 PSNR/SSIM에서 robustness·scale·protection·editability로 옮겨가는 게 분야 변곡점 신호예요.

📐 CV vs RO 대비

이번 주 키워드 분포를 CV/RO로 나눠 보면, 양쪽 다 한꺼번에 두꺼워진 결은 유지되지만 응축 축이 살짝 다시 갈라졌어요. CV는 diffusion(79) · multimodal(62) · rl(62) · benchmark(60) · vision-language(57) · segmentation(56) · reasoning(51) · efficient(50) · world(44) · robust(36)로 "생성·정렬·추론·벤치마크" 사각형이 어제 결을 유지하면서 world(44)가 한 칸 더 두꺼워졌고, RO는 manipulation(27) · rl(26) · vision-language(25) · vla(23) · navigation(22) · world(20) · agent(19) · robust(17) · reinforcement learning(17) · embodied(15)로 manipulation이 RO 최상단을 다시 차지하면서 vla(23)가 어제 17 → 23으로 +35% 점프했어요. 어제 등장한 "agentic robot" 어휘는 RO 19로 유지되고, "world" 어휘는 CV 44 / RO 20으로 둘 다 어제보다 한 칸 더 올라왔습니다 — WAM paradigm이 어휘 레벨에서도 두 분야 동시 침투를 가속하는 중이에요.

  • 공통 hot 키워드: rl(CV 62 / RO 26), vision-language(CV 57 / RO 25), world(CV 44 / RO 20), robust(CV 36 / RO 17), benchmark(CV 60 / RO 14) — 어제와 거의 동일 4축 + benchmark가 동시 응축. "RL로 정렬", "world 모델 어휘", "vision-language 인터페이스", "robust evaluation" 4축은 두 분야가 같이 두꺼워요.
  • CV에만: diffusion(79, RO의 10배 이상), segmentation(56), multimodal(62 vs RO 미진입), splatting(여전히 두꺼움), distillation(20대 유지) — 생성·렌더링·압축 어휘는 여전히 CV 전용 라인.
  • RO에만: manipulation(27, CV에 거의 안 잡힘), vla(23, +35% 가속), navigation(22), agent(19), reinforcement learning(17), embodied(15) — 물리·행동·agentic orchestration 어휘는 RO 전용.
  • 같은 단어 다른 맥락:
    • world (model) — CV에선 GTA처럼 image-to-3D world generation의 video-diffusion 기반 visual world simulator로 쓰이고, RO에선 MindVLA-U1·Causality-Aware E2E AD처럼 ego-centric joint scene modeling으로 action을 condition하는 planning latent로 쓰여요. 같은 어휘인데 한쪽은 "visual realism + asset generation", 한쪽은 "action-conditioned scene dynamics"로 평가축이 또 갈라집니다. 어제 짚었던 "predictive vs. plannable" 결이 오늘도 두 분야에서 같이 유지.
    • VLA — CV에선 MindVLA-U1의 driving E2E policy처럼 perception-to-action mapping으로 쓰이고, RO에선 BlockVLA·Realtime-VLA FLASH·AttenA+처럼 manipulation backbone의 inference engine·training infrastructure로 쓰입니다. CV는 "VLA가 무엇을 출력하는가", RO는 "VLA를 어떻게 deploy하는가"로 같은 약어 안에서도 질문 자체가 갈라져요. 오늘 두 결이 더 명확해졌습니다.
    • diffusion — CV에선 Asymmetric Flow Models·Pyramid Forcing의 generative model architecture로, RO에선 BlockVLA·Realtime-VLA FLASH·Test-time Sparsity for Action Diffusion처럼 정책 backbone의 inference cost를 깎는 도구로 쓰입니다. 한쪽은 perceptual quality 최적화, 한쪽은 control latency 최적화로 분기.

💡 오늘의 인사이트

1. VLA "inference engine" 라인이 AR vs Diffusion 두 paradigm을 동시에 가속하는 모드로 진입했다

어제(05-13) 보고드린 "VLA inference engine"이 thin-layer intervention 6종으로 막 형성된 sub-area였다면, 오늘은 그 라인이 한 칸 더 들어가 paradigm-축으로 분기합니다. BlockVLA는 autoregressive(AR) VLA의 sequential decoding latency를 block diffusion finetuning으로 깎고, Realtime-VLA FLASH는 diffusion-based VLA(dVLA)의 full-inference latency를 speculative inference framework로 깎아요. 두 paradigm이 정반대 방향에서 latency를 cut하는데 같은 날에 같이 깔린 게 결정적이에요 — 분야가 "어느 paradigm이 deploy 친화적인가"라는 fundamental 질문을 직접 던지기 시작한 거죠. 거기에 AttenA+가 "모든 action이 동등하게 informative하다"는 implicit assumption을 깨고 action inequality를 rectify하고, FrameSkip이 dense teleoperation의 균일 sampling이 데이터 낭비라는 진단으로 informative frame만 골라 학습하며, D-VLA가 high-concurrency distributed async RL framework를 깔고, What to Ignore, What to React가 RL fine-tuning의 deployment-time visual shift robustness를 다룹니다. inference·data·training 세 측면에서 동시에 두꺼워지는 신호 — 다음 1–2분기 안에 "VLA infrastructure"라는 명칭으로 sub-area 확정될 것 같습니다.

2. E2E Autonomous Driving이 VLA paradigm으로 본격 흡수되며 WOD-E2E human-driver baseline을 24시간 안에 두 번 초과했다

어제 보고드린 WAIM(World-Action Interactive Models)의 driving 도메인 instantiation(DAWN)이 24시간 안에 SOTA 갱신 + human baseline 초과까지 일사천리로 굳어졌습니다. MindVLA-U1은 unified streaming architecture로 VLA를 driving에 직접 가져오면서 WOD-E2E에서 8.20 RFS를 찍어 experienced human driver의 8.13 RFS를 처음 초과(2 diffusion steps · 16 FPS, 즉 throughput도 VA-class 유지), 같은 날 Driving Intents Amplify Planning-Oriented RL이 multi-intent GRPO로 single-trajectory continuous-action mode collapse를 깨고 best-of-128에서 9.14 RFS를 찍어 RAP(8.5)와 human(8.13)을 모두 큰 폭으로 넘어요. Action Emergence from Streaming Intent는 "action emergence" 자체를 target capability로 정식 정의해 평가축을 새로 깔고, Causality-Aware E2E AD는 ego-centric joint scene modeling으로 causal substrate를 깐 위에 prediction과 planning을 묶습니다. 같은 날에 method·SOTA·평가축·causal substrate 4축이 한꺼번에 채워지는 건 분야 변곡점의 전형적 패턴 — E2E driving이 modular pipeline에서 VA로, VA에서 VLA로 옮겨가는 두 번째 phase shift가 오늘 시작됐다고 봅니다. 다음 분기 ICRA·IROS·CVPR submission이 죄다 이 framing으로 쏠릴 가능성이 매우 큽니다.

3. 3DGS의 응축 axis가 "더 예쁘게(PSNR/SSIM)"에서 "실전 배포 친화(robust + scalable + protectable + editable)"로 옮겨갔다

이번 주 3D/Scene 버킷의 신규 GS 라인 6–7편이 모두 같은 결로 떨어진 게 인상적입니다. HarmoGS는 in-the-wild 캡처의 transient distractor와 illumination inconsistency를 conflict-aware gradient harmonization으로 풀고(image-level masking을 넘어 gradient-space 처방), RoSplat는 generalizable feed-forward GS의 input view 수와 해상도 변동에 대한 robustness를 정조준, BlitzGS는 city-scale 재구성을 system·algorithm·representation 3-level에서 active gaussian workload를 줄이는 distributed framework로 풀고, Z-Order Transformer는 feed-forward GS의 latency를 Z-order space-filling 기반으로 깎습니다. 거기에 GuardMarkGS가 3DGS asset의 ownership tracing과 instruction-driven edit deterrence를 동시에 잡고, Sparse Code Uplifting이 3D Language GS의 high-dim feature association을 효율화, Real2Sim이 physics-driven editable GS로 autonomous driving simulation을 만듭니다. 평가축이 PSNR/SSIM에서 robustness/scalability/protection/editability로 옮겨가는 분야 변곡점이에요 — 3DGS의 "research phase"가 끝나고 "production phase"가 시작되는 신호. 다음 분기 industry 응용 paper들이 폭증할 것 같습니다.

🔬 추천 연구주제

1. BlockVLA(AR→block-diffusion) vs Realtime-VLA FLASH(diffusion→speculative)의 cross-paradigm latency–success ablation

VLA inference engine 라인이 형성 중인 지금, 같은 backbone(예: OpenVLA / pi0 / RDT)에서 AR-paradigm BlockVLA와 dVLA-paradigm Realtime-VLA FLASH를 LIBERO·RoboTwin 2.0·real-robot 세 환경에서 latency–success 곡선으로 직접 비교하는 게 분야 미해결 질문에 첫 정량 답이 될 수 있어요. 구체적으론 (1) 동일 backbone+동일 dataset에서 두 paradigm을 each fine-tune, (2) 각 inference budget(10ms/30ms/100ms/300ms)에서 success rate 측정, (3) AttenA+의 action inequality rectification을 둘 다에 stack해 추가 gain을 측정, (4) FrameSkip의 data-side compression을 함께 ablation. 어제 추천드린 "VLA inference engine 6종 stackability" 라인의 자연스러운 paradigm-축 확장이고, 학회에서 항상 보기 좋은 latency-success Pareto frontier 그림을 만들 수 있어요. 인프라는 어제 라인과 90% 공유 가능 — ROI 매우 높습니다.

2. 3DGS "in-the-wild deployment kit"을 한 wrapper로 묶어 manipulation scene pipeline에 통합

오늘 등장한 4축 컴포넌트 — HarmoGS(in-the-wild robustness) + RoSplat(feed-forward generalization) + BlitzGS(scalable reconstruction) + GuardMarkGS(ownership protection)을 단일 wrapper로 묶고, 우리 lab의 manipulation scene을 in-the-wild 캡처 → 즉시 GS reconstruction → policy training의 closed-loop pipeline으로 만들면, 어제 추천드린 WAM #1(Forecast-GS 기반 World Predictor) 라인의 데이터 인프라를 동시에 해결할 수 있어요. 구체적 다음 단계로 (1) HarmoGS의 gradient harmonization을 manipulation의 hand-mounted camera 캡처에 적용해 illumination inconsistency 처리 가능성 측정, (2) RoSplat의 generalizable feed-forward를 새 manipulation scene에 zero-shot으로 적용해 reconstruction quality 측정, (3) BlitzGS의 distributed framework를 multi-room manipulation에 확장. 매우 실용적이면서도 분야 응축점을 정확히 짚은 라인입니다.

3. E2E driving VLA의 long-tail safety를 "belief-state residual risk" 평가 프레임으로 정량화

오늘 MindVLA-U1·Driving Intents Amplify·Causality-Aware E2E AD가 VA→VLA 전환의 method axis를 채웠는데 safety axis는 여전히 비어 있어요. 같은 날 등장한 Belief-Space Residual Risk와 Distributionally Robust Safety Filtering을 묶어 "VLA driving policy의 belief-conditioned residual risk"라는 첫 정량 프레임을 제시하면, 어제 보고드린 WAM survey의 evaluation column의 빈 칸을 정확히 메우는 1차 기여가 됩니다. 구체적으론 (1) MindVLA-U1을 backbone으로 두고 ego pose에 epistemic uncertainty를 inject, (2) Belief-Space Residual Risk의 framework로 long-tail scenario에서 risk profile 측정, (3) Distributionally Robust Safety Filter를 inference-time wrapper로 stack해 risk-bounded variant 만들기. WOD-E2E의 RFS 곡선은 평균 reward만 보여주는데, residual risk 프레임은 분포 tail을 직접 측정할 수 있어요. 다음 NeurIPS·ICLR·ICRA 한 사이클 분량의 main 후보로 충분합니다.

MindVLA-U1: VLA Beats VA with Unified Streaming Architecture for Autonomous Driving

arxiv:2605.12624 · Yuzhou Huang et al. CV/RO

핵심 주장: E2E autonomous driving이 modular pipeline → Vision-to-Action(VA) unification으로 이미 옮겨갔다면, VLA(Vision-Language-Action)는 그 다음 자연스러운 진화. 핵심 framing은 "language-side intent를 control signal로 명시적으로 받아 continuous trajectory generation을 condition하자"는 것. 저자들은 streaming action diffusion에 classifier-free guidance(CFG)를 결합해 action diffusion이 single batch update 안에서 intent에 따라 modulate되도록 unified streaming architecture를 제안합니다. WOD-E2E long-tail benchmark에서 8.20 RFS로 experienced human driver의 GT RFS 8.13을 처음으로 초과(2 diffusion steps만으로), prior VA/VLA SOTA 대비 planning ADE에서도 large margin으로 우위, 그리고 16 FPS throughput으로 RAP-DINO의 18 FPS와 거의 비등 — 즉 "throughput 손해 없이 quality에서 큰 폭 우위"라는 매우 강력한 trade-off 표를 갖고 옵니다. 어제(05-13) DAWN이 깐 WAIM framework의 driving 도메인 첫 SOTA-갱신 instance — 분야 변곡점입니다.

방법의 핵심: 의사코드로 표현하면, MindVLA-U1은 a_{1:H} ~ ε_θ(noise; vision, instruction_intent) 형태의 action diffusion에서 CFG term이 intent를 control signal로 명시 modulate. unified streaming의 핵심은 (1) vision · language · action을 단일 token stream으로 통합해 separate cascading branches를 없애고, (2) action diffusion step을 streaming inference의 thin layer로 두어 throughput을 VA-class로 유지하며, (3) intent CFG가 mode collapse(continuous-action policy가 한 demonstrated maneuver 주위에 cluster되는 현상)를 mode separation으로 깬다는 점이에요. "VLA beats VA" framing은 단순히 architecture 비교가 아니라 "language intent를 control signal로 받는 것이 RFS 같은 closed-loop quality metric에서 fundamental 우위를 가진다"는 분야적 주장입니다.

MindVLA-U1을 backbone으로 둔 채 우리 lab의 belief-state safety evaluation framework(Belief-Space Residual Risk + Distributionally Robust Safety Filter)를 inference-time wrapper로 stack하면, "WOD-E2E human baseline을 넘은 VLA driving policy의 belief-conditioned residual risk"라는 첫 정량 프레임이 만들어져요. 어제 추천드린 WAM survey의 evaluation column 공백 채우기와 직접 연결되고, 다음 NeurIPS/ICRA 사이클 main 후보. 단기적으론 MindVLA-U1의 공개 checkpoint 또는 re-implementation을 우리 driving simulator(CARLA·nuScenes minivariant)에 띄우고 RFS 대신 collision rate · lane adherence · intervention rate 3축으로 cross-evaluate하는 게 첫 단계입니다.

BlockVLA: Accelerating Autoregressive VLA via Block Diffusion Finetuning

arxiv:2605.13382 · Ruiheng Wang et al. RO

핵심 주장: 어제 보고드린 "VLA inference engine" 라인이 thin-layer intervention(A3, GridS, Premover 등)으로 막 형성된 sub-area라면, BlockVLA는 그 라인이 paradigm-level로 한 칸 들어간 결정적 사례. AR(autoregressive) VLA가 token-by-token sequential decoding 때문에 inference latency가 본질적으로 길다는 fundamental 한계를, AR backbone을 그대로 두면서 fine-tuning 단계에서 block diffusion으로 decoding scheme을 바꿔 가속하는 게 핵심. AR 학습의 reasoning 강점을 유지한 채 inference만 block-parallel로 만들어내요. 같은 날 등장한 Realtime-VLA FLASH가 정반대 paradigm(dVLA → speculative inference)에서 같은 latency-cut 목표를 추구한다는 점에서 두 논문은 mirror image — 분야가 "AR vs Diffusion 어느 paradigm이 deploy 친화적인가"라는 fundamental 질문을 직접 던지기 시작한 신호입니다.

방법의 핵심: AR-VLA의 action token sequence를 길이 B의 block으로 나누고, 각 block 안에서는 diffusion-style parallel decoding을 수행하되 block 간에는 AR ordering을 유지. 의사코드로는 for b in 1..N_blocks: a_{(b-1)B+1 : bB} ~ ε_θ(noise; a_{1:(b-1)B}, vision, instruction) 형태. 핵심은 (1) AR pretraining의 reasoning을 유지하면서 (2) block 내 sequential cost를 diffusion parallel로 cut, (3) block 경계에서 AR consistency를 유지해 long-horizon coherence를 잃지 않는다는 점이에요. fine-tuning 단계에서만 손을 대고 pretraining은 그대로 두는 게 BlockVLA의 매력 — 기존 OpenVLA·RDT·pi0 같은 강한 AR backbone에 plug-and-play로 적용 가능.

약점·한계: 첫째, block size B의 선택이 trade-off의 핵심인데 task-dependent — long-horizon manipulation에서 B를 크게 잡으면 latency는 cut되지만 block 경계에서 coherence가 깨질 risk. abstract만으론 B-sensitivity가 명확하지 않아 본문에서 ablation 확인 필수. 둘째, AR pretrain의 reasoning을 "유지한다"는 주장이 매력적이지만 block-diffusion fine-tuning이 reasoning을 sub-block level로 fragment할 가능성 — long-horizon causal chain이 필요한 task(예: 다단계 manipulation)에서 retain 정도가 본문 확인 지점. 셋째, 정반대 paradigm Realtime-VLA FLASH와의 head-to-head가 같은 날 논문이라 본문에 들어가지 않았을 가능성 — 두 paradigm의 cross-comparison은 우리가 직접 해야 할 빈 자리(추천주제 #1). 넷째, "block diffusion finetuning"이 standard diffusion FT보다 안정적인지에 대한 training dynamics ablation이 본문에서 어떻게 다뤄지는지 핵심.

우리 랩 파이프라인에의 1차 영향: 위 추천주제 #1과 직결. 우리 lab의 manipulation pipeline에서 BlockVLA를 한 paradigm representative로, Realtime-VLA FLASH를 다른 paradigm representative로 두고 동일 backbone·동일 dataset에서 latency-success Pareto frontier를 측정하면, "VLA inference engine"의 paradigm-level 첫 정량 비교가 만들어집니다. 어제 추천드린 thin-layer intervention 6종 stackability 라인과 80% 인프라 공유 가능. 단기적으론 BlockVLA의 block size B를 LIBERO·RoboTwin 2.0 각 task에서 sweep해 task-specific optimal block size를 찾는 게 첫 단계입니다.

BlitzGS (city-scale 3DGS) — cherry-pick 의심 + 비공개 baseline 가능성. "city-scale at lightning speed" 라는 강력한 framing이지만 city-scale 3DGS의 prior baseline(CityGS, VastGaussian, GauU-Scene 등)과의 head-to-head가 abstract만으론 명확하지 않음.
